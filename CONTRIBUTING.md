# 콘텐츠 작성 가이드 (비개발자용)

이 문서는 **개발 지식 없이도** 공지사항이나 릴리즈 노트를 작성·수정하는 방법을 안내합니다.

코드를 만질 필요 없이 **GitHub 웹 페이지에서 모두 처리**할 수 있어요.

---

## 📢 공지사항 작성

### 1. 폴더 진입

브라우저에서 repo의 `docs/announcements/` 폴더로 이동합니다.

### 2. 새 파일 생성

오른쪽 상단의 **`Add file`** → **`Create new file`** 클릭

### 3. 파일명 작성

파일명 규칙: **`YYYY-MM-DD-짧은-제목.md`**

예시:
```
2026-05-01-new-feature.md
2026-05-15-maintenance-notice.md
```

### 4. 내용 작성

아래 형식을 그대로 복사해서 사용하세요:

```markdown
---
title: 여기에 공지 제목을 입력하세요
category: NEW
date: 2026-05-01
---

# 여기에 공지 제목을 입력하세요

여기에 본문을 작성합니다.

## 소제목

- 항목 1
- 항목 2

자세한 내용은 [관련 페이지](https://...) 를 참고하세요.
```

### 5. category 종류

| category | 사용 시점 |
|---|---|
| `NEW` | 새 기능 출시, 신규 서비스 안내 |
| `공지` | 점검, 일반 공지사항 |
| `가이드` | 사용법 안내, 가이드 업데이트 |

### 6. 저장

페이지 하단의 **`Commit changes...`** 버튼 클릭 → 다시 **`Commit changes`** 클릭

➡️ 1~2분 후 Hub 페이지에 자동 반영됩니다.

---

## 📋 릴리즈 노트 작성

### 1. 폴더 선택

서비스에 맞는 폴더로 이동:

- **Code Mate** → `docs/releases/code-mate/`
- **Code Mate with OpenCode** → `docs/releases/opencode/`
- **Code Mate with Roo** → `docs/releases/roo/`

### 2. 새 파일 생성

파일명 규칙: **`v버전번호.md`**

예시: `v2.1.0.md`, `v1.4.2.md`

### 3. 내용 작성

```markdown
---
version: v2.1.0
date: 2026-04-29
features:
  - 첫 번째 주요 기능 (홈에 노출됨)
  - 두 번째 주요 기능 (홈에 노출됨)
  - 세 번째 주요 기능 (홈에 노출됨)
---

# Code Mate v2.1.0

## ✨ 주요 기능

### 첫 번째 주요 기능

상세 설명을 적습니다.

## 🐛 버그 수정

- 수정된 버그 1
- 수정된 버그 2
```

### 4. 중요한 점

- `features:` 항목에 적은 내용이 **홈 페이지의 카드에 표시**됩니다
- 처음 3개만 홈에 보이므로 **가장 중요한 것 3개**를 위에 배치하세요
- 제목 위 `---` 사이의 부분(frontmatter)은 **반드시 형식대로** 작성해야 합니다

---

## 🖼️ 기존 글 수정

1. 수정하려는 `.md` 파일 클릭
2. 오른쪽 상단의 ✏️ **연필 아이콘** 클릭
3. 내용 수정
4. **`Commit changes...`** → **`Commit changes`** 클릭

---

## ❌ 글 삭제

1. 삭제하려는 `.md` 파일 클릭
2. 오른쪽 상단의 🗑️ **휴지통 아이콘** 클릭
3. **`Commit changes...`** 클릭

---

## ⚠️ 주의사항

### 형식 깨뜨리지 않기

`---` 로 둘러싸인 상단 영역(frontmatter)이 가장 중요해요. 이 형식이 깨지면 홈 페이지에 표시되지 않습니다.

❌ 잘못된 예:
```
title 제목
date 2026-04-29
```

✅ 올바른 예:
```
---
title: 제목
date: 2026-04-29
---
```

### 날짜 형식

반드시 **`YYYY-MM-DD`** 형식으로:
- ✅ `2026-04-29`
- ❌ `2026.04.29`, `2026/4/29`, `26-4-29`

### 미리보기

작성 중에 **`Preview`** 탭을 누르면 어떻게 보일지 확인할 수 있습니다.

---

- 문제가 있으면 운영자에게 문의해주세요
